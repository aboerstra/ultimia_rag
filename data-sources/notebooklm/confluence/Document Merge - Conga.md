It:</p><ol start="1"><li><p>Connects to the appropriate Salesforce server.</p></li><li><p>Uses the current Opportunity's ID to gather data.</p></li><li><p>Passes Account ID and Dealer ID as filters to multiple reports.</p></li><li><p>Selects a Conga template dynamically based on a field.</p></li><li><p>Opens a template selection window.</p></li><li><p>Does not zip the generated file.</p></li><li><p>Automatically opens the generated document.</p></li></ol><h3>Breakdown of Logic</h3><p><strong>Base URL and Composer Endpoint</strong></p><p>When clicked, it tells Salesforce to call the Conga Composer Visualforce page, which is part of the Conga Composer managed package. </p><p><strong>Server URL</strong></p><p>This parameter specifies the Salesforce API server URL that Conga Composer will connect to.<br />The merge field <code>{!API.Partner_Server_URL_370}</code> dynamically pulls in the correct server URL.</p><p><strong>Opportunity ID</strong></p><p>This specifies the Salesforce record ID (in this case, the Opportunity ID) that Conga will use to generate the document.</p><p><strong>Report ID and Parameter Values</strong></p><p>    ReportId Parameter: The report ID (<a href="https://maximcc.lightning.force.com/lightning/r/Report/00OF0000004AWQp/edit?queryScope=userFolders">00OF0000004AWQp</a>) = Account with Contact Roles Report - gets the Account (from the Opportunity) and the related Contacts. WHY???</p><p>    pv0 Parameters: These are parameter values (filters) passed to the report.</p><p>The syntax is a bit odd, but essentially it is passing filters to multiple reports:</p><p>    Report  <a href="https://maximcc.lightning.force.com/lightning/r/Report/00OF0000002eP59/view?queryScope=userFolders">00OF0000002eP59 </a>Gets Vehicle data from the Opportunity</p><p>Report <a href="https://maximcc.lightning.force.com/lightning/r/Report/00OF0000002eSZEMA2/view">00OF0000002eSZE</a> Gets the Opportunity.Dealer&rsquo;s Address</p><p>This setup allows Conga to pull data from multiple reports and dynamically filter them using the Opportunity&rsquo;s Account ID and Dealer ID.</p><p><strong>Template ID </strong></p><p>This specifies the Conga template ID that will be used to generate the document.
